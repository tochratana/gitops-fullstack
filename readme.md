# рҹҡҖ Fullstack Application вҖ” GitOps Repository

A GitOps-style Helm chart repository for deploying a fullstack application to Kubernetes via **ArgoCD**.  
The stack includes a **Next.js** frontend, **Spring Boot** backend, and **PostgreSQL** database, all managed as an umbrella Helm chart.

## вҡҷпёҸ How It Works

### 1. Helm Umbrella Chart Pattern

`full-stack-app` is the **umbrella (parent) chart**. It declares `frontend`, `backend`, and `postgres` as dependencies in `Chart.yaml`.

When you run `helm dependency update`, Helm packages each sub-chart into a `.tgz` file and places them in `charts/full-stack-app/charts/`. These are the files ArgoCD uses.

### 2. ArgoCD Renders YAML From Your Repo

When ArgoCD syncs, it runs `helm template` internally:

```
Git Repo
  в””в”Җв”Җ charts/full-stack-app/
        в”ңв”Җв”Җ Chart.yaml
        в”ңв”Җв”Җ values.yaml
        в””в”Җв”Җ charts/*.tgz         вҶҗ sub-charts (compressed)
              вҶ“
       helm template (by ArgoCD)
              вҶ“
    Kubernetes Manifests (.yaml)
    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
    в”Ӯ  Deployment (frontend)   в”Ӯ
    в”Ӯ  Deployment (backend)    в”Ӯ  вҶ’ Applied to Kubernetes
```

## рҹӣ пёҸ Local Development

### Prerequisites

- `kubectl` вҖ” Kubernetes CLI
- `helm` (v3+) вҖ” Helm CLI
- `git`
- Access to a Kubernetes cluster
- ArgoCD installed on the cluster
- Nginx Ingress Controller
- `cert-manager` (for TLS / Let's Encrypt)

### First Time Setup

```bash
# 1. Clone the repo
git clone https://github.com/your-org/gitops-fullstack.git
cd gitops-fullstack

# 2. Package sub-charts into .tgz (REQUIRED before pushing)
helm dependency update ./charts/full-stack-app

# 3. Dry-run to verify templates render correctly
helm template my-release ./charts/full-stack-app \
  -f environments/dev/values.yaml

# 4. (Optional) Lint the chart
helm lint ./charts/full-stack-app -f environments/dev/values.yaml
```

## вҡЎ Auto Push Script (`push.sh`)

`push.sh` automates the full workflow in one command:
1. Runs `helm dependency update` (refreshes `.tgz` files)
2. Stages all git changes
3. Commits with a message (auto-timestamped if not provided)
4. Pushes to origin

### Usage

```bash
# Make executable (first time only)
chmod +x push.sh

# Push with default timestamped commit message
./push.sh

# Push with a custom commit message
./push.sh "feat: update frontend image tag to v1.2.3"
```

After a successful push, ArgoCD will automatically detect the change and begin syncing.

## рҹ”§ Updating Image Tags

Image tags are defined in `charts/full-stack-app/values.yaml`:

```yaml
images:
  frontend:
    repository: tochratana/prod-ui
    tag: "b2518c6c"        # вҶҗ Change this to deploy a new frontend image
  backend:
    repository: tochratana/prod-api
    tag: "01d931c2"        # вҶҗ Change this to deploy a new backend image
```

After editing, run `./push.sh "update image tags"` and ArgoCD will roll out the new pods.

## рҹ”‘ Secrets

Sensitive values (DB passwords etc.) should be stored as Kubernetes Secrets **outside this repo** (e.g., via Sealed Secrets, External Secrets Operator, or manually created).

The chart references:
- `postgres-secret` вҖ” must be created manually in the target namespace before first sync

```bash
kubectl create secret generic postgres-secret \
  --from-literal=postgres-password=<your-password> \
  --from-literal=password=<your-password> \
  -n fullstack
```

## рҹ“Ӣ Troubleshooting

| Problem | Solution |
|---|---|
| ArgoCD shows `ComparisonError` | Run `helm dependency update ./charts/full-stack-app` and push |
| `helm template` fails with exit 1 | Run `helm lint ./charts/full-stack-app` to find template errors |
| Pods not starting | Check `kubectl describe pod <name> -n fullstack` |
| Ingress not working | Verify nginx-ingress and cert-manager are installed |
| `.tgz` files missing | Run `helm dependency update ./charts/full-stack-app` |
